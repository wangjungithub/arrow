OAuthAutomator

(
  • config
)

Defined in lib/server/oauth-automator.js:91

Parameters:

  • config String | Object

    [config] A confihuration object. If a string type is given, it should be the path of the configuration file(json) otherwise should an object with configuration elements.

    Valid configuration object properties:

    • [configPath] String optional

      A file path of configuration file. If all configurations are contained in the file, the file path can be directly passed to this constructor instead of placing it to this configuation object. If the same property is also given in this configuration object, the one in the file will be overriden.

    • [timeoutForTokenGeneration] Number optional

      29(default)

    • [wsUrl] String optional
    • [wsMethod] String optional

      "GET"(default)

    • [wsQueryParams] String optional
    • [signatureMethod] String optional

      "HMAC-SHA1"(default)

    • [oauthProvider] String optional

      "yahoo"(default)

    • [yahooAuthVersion] String optional

      "v1" or "v2"(default)

    • [oauthType] String optional

      "external"(default), keep "internal" for Yahoo! internal use.

    • [externalOAuthScope] String optional

      Used to specify external OAuth scope.

    • [oauthServer] String optional

      An OAuth provider URL

    • [oauthVersion] String optional

      Either "1.0" or "1.1". Default is "1.0".

    • [oauthMode] String optional

      Only "header"(default) is available. The "url" is broken.

    • [consumerKey] String optional

      OAuth consumer key

    • [consumerSecret] String optional

      OAuth consumer secret

    • [appId] String optional

      OAuth application id.

    • [ExtAllConsumerKey] String optional

      This is an example property name for the "external" type when "externalOAuthScope" is given and its value is "All". Ext+"{externalOAuthScope}"Consumer[AppId | Key | Secret] will be used for the given external scope.

    • [ExtAllConsumerSecret] String optional

      This is an example property name for the "external" type when "externalOAuthScope" is given and its value is "All". Ext+"{externalOAuthScope}"Consumer[AppId | Key | Secret] will be used for the given external scope.

    • [ExtAllConsumerAppId] String optional

      This is an example property name for the "external" type when "externalOAuthScope" is given and its value is "All". Ext+"{externalOAuthScope}"Consumer[AppId | Key | Secret] will be used for the given external scope.

    • [needUserCred] Boolean optional

      true(default) or false. If true, 3-legged protocol will be used.

    • [username] String optional

      Username. Required if needUserCred is set to "true".

    • [password] String optional

      Password. Required if needUserCred is set to "true".

generateOAuth

(
  • attrs
  • [cb]
)

Defined in lib/server/oauth-automator.js:262

Parameters:

  • attrs Object

    (Optional) An object with configurations(attributes). If this object is given, attributes determined in the constructor will be overriden. The updated attributes will remain even after this method is completed. All attributes except 'configPath' are allowed here.

  • [cb] Function optional

    A callback function to be called once authrization credentials are generated.

    • [err] Error optional

      An error object if auth credential generation is failed.

    • [headers] Object optional

      An object with generated auth headers. Example: {"Yahoo-App-Auth": "...", "Authorization": "..."}

    • [token] Object optional

      An object with access token data for the external type. The access token object has 5 properties; oauthToken, oauthTokenSecret, oauthTokenExpires, oauthTokenSessionHandle, oauthTokenSessionHandleExpires. See the OAuth spec for details.
